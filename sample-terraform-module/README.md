# Sample Terraform Module

End-to-end reference of the layout the [`hcl-module-test.yml`](../.github/workflows/hcl-module-test.yml) Terratest pipeline expects from an ACAI HCL module. Copy or mirror this structure when bootstrapping a new module.

See the full pipeline reference in [`docs/terratest.md`](../docs/terratest.md).

## Layout

```
sample-terraform-module/
├── README.md
├── main.tf                          # module under test
├── variables.tf
├── outputs.tf
├── examples/                        # default: hcl_terratest_examples_path = "examples"
│   └── complete/
│       ├── backend.tf               # partial S3 backend (values from backend.json)
│       ├── provider.tf              # aws providers + aliases
│       ├── variables.tf             # filled from <TEST_BED>_TESTBED_TFVARS
│       ├── main.tf                  # required_providers + module instantiation
│       └── outputs.tf               # `test_success*` assertion outputs
└── tests/                           # default: hcl_terratest_path = "tests/terratest"
    └── terratest/
        ├── complete_test.go         # default test (runs examples/complete)
        └── helpers_test.go          # latest helpers
```

## File responsibilities

### Module root

| File           | Purpose                                                                                            |
|----------------|----------------------------------------------------------------------------------------------------|
| `main.tf`      | The module under test. Pins `required_version` / `required_providers` and declares its resources. |
| `variables.tf` | Module inputs.                                                                                     |
| `outputs.tf`   | Module outputs (consumed by examples and downstream callers).                                      |

### `examples/complete/` — anatomy of a test example

| File           | Responsibility                                                                                                       |
|----------------|----------------------------------------------------------------------------------------------------------------------|
| `backend.tf`   | `terraform { backend "s3" {} }` — concrete values injected via `backend.json` from `<TEST_BED>_TESTBED_BACKEND_JSON`.|
| `provider.tf`  | All `aws` providers / aliases the module needs; role ARNs templated from `var.account_ids`.                          |
| `variables.tf` | Variables fed by `testbed.tfvar` (= `<TEST_BED>_TESTBED_TFVARS`): account IDs, region, partition, role name.         |
| `main.tf`      | Pins `required_version` / `required_providers`; instantiates the module under test with `source = "../../"`.         |
| `outputs.tf`   | One or more `test_success*` outputs evaluating to the boolean literal `true` on a healthy deployment.                |

### `tests/terratest/` — anatomy of the Go test

| File              | Responsibility                                                                                                |
|-------------------|---------------------------------------------------------------------------------------------------------------|
| `complete_test.go`| `TestExampleComplete` deploys `examples/complete`, asserts `test_success == "true"`, destroys the stack.      |
| `helpers_test.go` | `loadBackendConfig` (per-test state key), `getHclBinary` (Terraform / OpenTofu via `TERRATEST_TERRAFORM_BINARY`), `outputClean` / `outputMapClean` / `outputRawClean` (warning-stripping output readers). |

`go.mod` and `go.sum` are generated by the workflow on first run; you do not need to commit them.

## How to use this sample

1. Copy the whole folder into a new repository (or mirror the layout into an existing one).
2. Replace the contents of `main.tf` / `variables.tf` / `outputs.tf` with your real module.
3. Adjust `examples/complete/main.tf` to pass the inputs your module requires and rewire the `providers = { ... }` block to match the aliases declared in `provider.tf`.
4. Express assertions as `test_success*` outputs in `examples/complete/outputs.tf`.
5. Wire up the workflow in your repo (see [`docs/terratest.md`](../docs/terratest.md#minimal-consumer-workflow)).
